# 01_ursula_dataset: RL Gym Environment for DSP Parameter Learning

## Overview
Ursula learns to map physical metric differences to optimal DSP parameters through Reinforcement Learning. This pipeline generates the Gym environment and training data on Kaggle.

## Core Concept
Ursula does not learn from labeled (X → Y) pairs. She learns by **exploring** — proposing DSP parameters, observing how they affect the audio, and receiving a reward based on how close the result's metrics get to the reference.

**The RL Loop per episode:**
1. Environment samples a (degraded_audio, reference_audio) pair
2. Ursula (the agent) proposes DSP parameters
3. Environment applies those parameters to the degraded audio
4. Environment extracts metrics from the processed result
5. Reward = -MSE(M_result, M_ref)
6. Ursula updates her policy based on the reward

## Pipeline Steps

### Step 1: Source Audio Preparation
- Mount pristine vocal datasets (VCTK, LJSpeech, DAPS cleanraw)
- Sterilize: noise-reduce, trim silence, normalize to -23 LUFS
- Segment into 5s clips
- Create reference pool: run a subset through **Resemble-Enhance** for idealized target audio

### Step 2: Degradation Suite (Headless C++ Plugins)
To simulate the "Input Audio" side of the pair, apply randomized DSP effects:
- Parametric EQ (random band gain / Q factor / center frequency)
- Digital clipping / soft saturation
- Broadband noise injection (white, pink, brown, HVAC hum)
- Comb filtering (1-10ms micro-delays for room simulation)
- Bitcrushing / quantization
- Highpass / lowpass filtering
- Gain jitter
- Convolution with "bad" IR (optional)

Each degraded clip is paired with its Resemble-Enhance reference to form an episode pair.

### Step 3: Metrics Extraction (Portable)
Extract physical metrics from both degraded and reference audio:
- **LTAS (64D):** Bark-scale energy distribution via STFT
- **LUFS (1D):** ITU-R BS.1770-4 integrated loudness
- **Dynamic Range (2D):** Crest factor + zero-crossing rate

### Step 4: Gym Environment Definition
The Gym environment wraps the DSP chain and metrics extraction:

```
Observation Space: (M_current, M_ref) — 67D concatenated
Action Space: Continuous — EQ gains, comp threshold/ratio, makeup gain
Reward Function: r = -w1 * MSE_LTAS - w2 * MSE_LUFS - w3 * MSE_Dyn
Termination: Episode ends when MSE < threshold or max steps reached
```

The headless C++ plugins are the "plant" that Ursula controls through actions.

### Step 5: Serialization (for Pre-Generated Episodes)
If running off-policy or pre-generating experience:
- Save each transition as: `{deg_audio, ref_audio, M_deg, M_ref, action, reward, next_M}`
- Format: Parquet or JSONL, ~19 GB budget

## Output
A Gymnasium-compatible environment that Kaggle notebooks can instantiate and use to train Ursula via SAC/PPO.
